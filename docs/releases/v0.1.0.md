# v0.1.0 — core loop, CI-verified, pre-validation

> The deterministic root-cause engine, its bounded LLM explanation layer,
> the CLI, the web UI, and the full CI/deploy pipeline — built to depth,
> tested end to end, and honest about what isn't done yet.

**Culprit** answers the most expensive question in an incident — *"which
recent change caused this?"* — by correlating deploys, config, Terraform,
and Kubernetes events against the alert. The correlation is deterministic
and cited; an LLM only *explains* on top, bounded so it provably cannot
invent a conclusion.

This is a **pre-1.0 engineering milestone, not a production claim.** No
real-customer validation has happened yet — that's the deliberately-next
step, and the repo says so everywhere.

## Highlights

- **One core loop, done deeply** — Detect → Correlate → Reason → Recommend
  → Human-approve → Learn. Every step before the LLM is pure, deterministic,
  tested code.
- **The LLM is not the source of truth** — a Rule Engine + Knowledge Graph
  produce the verdict with cited evidence; the LLM's confidence adjustment
  is hard-capped at ±15%, and every fact it cites is validated against real
  evidence or stripped. The system gives a correct answer with the LLM off.
- **precision@1 = 100%** on an 18-scenario golden set, gated in CI — a rule
  or weight change that flips any ranking fails before merge.
- **Review-hardened** — shipped through a full Staff-level review: input
  validation at the file boundary, guarded external API calls, deterministic
  connection lifecycle.

## What's included (built & tested)

| Area | Detail |
|---|---|
| Incident Simulation Harness | 18 ground-truthed scenarios spanning cross-service config, deadlocks via shared DB, IAM/Terraform, feature flags, DNS/mesh/network policy, and monitoring-topology failures |
| Correlation engine | Knowledge Graph (dependency + sibling + `monitored_by` coupling) and a 5-rule engine, each rule individually evidenced |
| Confidence scoring | Composite `rules · history · LLM` with the ±0.15 LLM bound |
| RAG incident memory | Lexical and pgvector backends, both tested against a real Postgres in CI; two-sided precedent guardrail |
| LLM explanation layer | Bounded reasoning behind an evidence-grounding guardrail; degrades to the deterministic verdict on any failure |
| `culprit` CLI | `demo` / `diagnose` / `learn` / `eval`, with persistent memory and a backend-comparison mode |
| Web UI | Next.js Incident List + Detail, rendered from real pipeline output |
| Deploy | Docker image + Helm chart, verified on a live `kind` cluster in CI; EKS/ECR/OIDC Terraform, `validate`-clean |
| CI/CD | GitHub Actions: golden-set eval gate, real-Postgres tests, kind deploy, Terraform validate |

## Deliberately *not* in this release

Gated on credentials or human work — not code, and not claimed as built:

- Live LLM narration and semantic embeddings (need `ANTHROPIC_API_KEY` /
  `VOYAGE_API_KEY`; wired and contract-tested with deterministic stand-ins).
- EKS `terraform apply` (written and `validate`-clean; needs AWS credentials).
- OpenTelemetry tracing, Prometheus/Grafana self-observability, and the REST
  API server (designed in `docs/`, not built — the CLI is the interface).
- **Any real-customer validation** — the Phase 0 exit criterion, still open.

## Verification

- **55 tests pass locally**, plus a **14-test Postgres/pgvector suite** against
  a real database in CI (69 total). CI green on the tagged commit.
- The regression suite *is* the golden-set evaluation: precision@1, per-layer
  and per-rule metrics, plus the LLM guardrail contract.
- `culprit eval` publishes the metrics report — including an honest
  authored-data-bias flag when a single rule matches the whole pipeline on
  the simulated set.

## Try it (no credentials)

```
cd services/correlation-engine
python -m correlation_engine.cli demo deadlock     # a real verdict with cited evidence
python -m correlation_engine.cli eval              # the golden-set metrics
```
